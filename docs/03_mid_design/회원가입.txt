sequenceDiagram
  actor User as User 
  participant Web_Server as 웹 서버
  participant 사용자 인증 as 사용자 인증 모듈
  participant 회원 데이터베이스 as 데이터베이스(등록된 사용자)
  User ->> Web_Server: 1. 회원가입 요청 (ID)
  Web_Server ->> 사용자 인증: 2. 사용자 등록 요청 (ID)
  사용자 인증 ->> 회원 데이터베이스: 3. 사용자 존재 여부 확인 (ID)
  alt 계정 생성이 가능한 경우 (ID 중복 없음)
    회원 데이터베이스 -->> 사용자 인증: 4. 사용자 정보 (없음)
    사용자 인증 ->> 회원 데이터베이스: 5. 신규 사용자 정보 저장 요청 (ID)
    회원 데이터베이스 -->> 사용자 인증: 6. 저장 성공 응답
    사용자 인증 -->> Web_Server: 7. 등록 성공 응답
    Web_Server -->> User: 8. 회원가입 성공 페이지/메시지 표시
  else 계정 생성이 불가능한 경우 (ID 중복)
    회원 데이터베이스 -->> 사용자 인증: 4.a 사용자 정보 (있음)
    사용자 인증 -->> Web_Server: 7a. 등록 실패 응답 (사유: ID 중복)
    Web_Server -->> User: 8a. 회원가입 실패 메시지 표시 (ID 중복)
  end
